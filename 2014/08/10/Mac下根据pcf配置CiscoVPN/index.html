Mac下根据pcf配置Cisco VPN

发表于 2014-08-10   |   分类于 Mac OS系列   |  

背景

合作方给了内网的VPN，同时给了pcf文件以及Cisco.VPN.Client.for.Windows ，在windwos下连接成功。下面记录下自己在MAC下连接的方法。

纯记录步骤

1、首先调研发现了，Cisco 走的是 IPSec 方式。同时在 “系统偏好设置“——“网络”——“+”——“VPN”——“VPN类型中”找到了Cisco IPSec（喜，看来是能搞定~）

2、新建该类型VPN ，服务名称随便写
3、接下来，我们需要的信息有：

  • 服务器地址
  • 账户名称
  • 密码
  • （鉴定设置中）共享的密钥
  • （鉴定设置中）群组名称
    4、OK,直接说结果，打开pcf，我们能找到很多字段。
  • 服务器地址： Host= xx.xx.xx.xx
  • 账户名称： pcf中貌似没有，是管理员给你的用户名
  • 密码：pcf中貌似没有，是管理员给你的密码
  • （鉴定设置中）共享的密钥：’GroupPwd= ‘ ,如果pcf中该字段为空，请见第5条
  • （鉴定设置中）群组名称：GroupName=xxxxx
    5、如果上一部中GroupPwd字段为空，且有类似如下字段：

说明Group密码是被加密过的，我们可以在该网站进行解密
http://www.unix-ag.uni-kl.de/~massar/bin/cisco-decode
感兴趣还可以看看解密方法的源码~ cisco-decrypt.c
6、解密过后，把密码填进去就OK了. “应用” —— “连接” 。 然后试下应该可以ping通内网的机器了哦~

后续

问题1 ： 连上后，貌似我的终端里用户名被改为 “bogon”了。。断掉依然是。。

bogon是指那些不该出现在internet路由表中的地址
“系统偏好设置”-》“网络” -》 “以太网”或“wifi” -》 “高级…”-》WINS 里面就能看到这个”bogon”
这是因为终端会先向 DNS 请求查询当前 IP 的反向域名解析的结果，如果查询不到再显示我们设置的计算机名。而由于我们的 DNS 错误地将保留地址反向的 NS 查询结果返回了 bogon. 其中 bogon 本应该用来指虚假的 IP 地址，而非保留 IP 地址。因此就出现了会时不时地打印 bogon 这种奇怪名字作为计算机名的现象了。那么如何让终端只显示我们想要的计算机名而不总是从 DNS 返回结果呢？

解决方案：
在终端中执行以下命令即可（需要输入一次管理员密码）
sudo hostname your-desired-host-name
sudo scutil —set LocalHostName $(hostname)
sudo scutil —set HostName $(hostname)
