centos安装utorrent

发表于 2014-09-13   |   分类于 linux记录   |  

接上一篇在一台旧电脑上安装好centos 之后，躁动的心情促使着我们又开始折腾啦———— 能不能安装上utorrent呢，在Mac上挂BT也不方便啊。

调研阶段

在Linux里想下载bt，通常需要Web界面。已知的选择有几个：

  • utorrent
  • vuze
  • transmission

一直用的就是utorrent，先拿他来开刀。

utorrent

到utorrent官网下载Linux 版本的。
里面有 Debian 和 Ubuntu 版本的，木有CentOs 囧。

拿Ubuntu得凑合用吧！

额，好吧，这里的很奇怪我wget 下来的是一个Html。我最后用浏览器下载，然后scp传到服务器的

注意注意，前方大坑预警。

这里有几种情况：

  • error while loading shared libraries: libssl.so.X.X.X: cannot open shared object file: No such file or directory.

    这种情况，是因为openssl的版本不对啦。现在新版的utorrent 貌似都是依赖1.0.0版本了（额，网上资料大部分还是依赖0.9.8）的。确实1.0.0的童鞋，可以去安装一个1.0.0版本的openssl啦，不再赘述。
    网上遇到的很多解决方法是：把已有的版本给需要的版本做个软链。

    注意，如果是64位系统，上面路径是 /usr/lib64/xxxxxx

    • /usr/lib64/libcrypto.so.1.0.0: no version information available (required by ./utserver)

      这种情况，这种情况只是个Waring.. 没有其他BUG的情况下是没有影响的。

    • libc.so.6: version `GLIBC_2.14’ not found

      这个事情不太好办。glibc是gnu发布的libc库，遇到这种情况通常是我们系统的glibc版本较低。详情见我新开的一贴吧,我主要参考的是这里.

      这些问题解决了之后。执行./utserver
      然后访问”localhost:8080/gui” ，如果是远程的服务器的话是”xxxx:8080/gui” xxxx 为服务器IP地址。初始账号为admin 密码为空，就能看到我们熟悉的页面啦~

iptables 配置

如果仍然无法打开，可能是防火墙问题。
可以先尝试关闭iptables服务

确定是防火墙问题后，我们可以给防火墙加入规则：开启8080端口

建议使用 -I 而不是 -A （被这个问题坑过）
